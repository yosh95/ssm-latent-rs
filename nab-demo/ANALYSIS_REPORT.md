## 2. 解決済みの課題（根本原因の解消）

```
Layer 5: スコアリング（パーセンタイル変換） ─→ [解決] スコアが適切に分布し、NAB閾値最適化が機能
Layer 4: EMA/MAD キャリブレーション ───────→ [解決] 試用期間による安定化と汚染防止を実装
Layer 3: 推論パイプライン（Streaming） ─────→ [解決] forward_step() によりチャンク境界の偽スパイクを排除
Layer 2: 学習パイプライン ───────────────────→ [進行中] 50%データ、100epoch、安定したLR
Layer 1: アーキテクチャ ──────────────────────→ [次段階] マルチスケール層、特徴量拡張
```

### 改善効果の内訳
1.  **隠れ状態の連続性 (Layer 3)**: 旧実装では 5000 ステップごとに状態がリセットされ、膨大な FP（偽陽性）を発生させていました。ストリーミング推論の導入により、これがゼロになりました。
2.  **ロバストな標準化 (Layer 4)**: 異常値に引きずられない MAD 定数と汚染防止ロジックにより、異常検知直後の感度低下を抑制しました。
3.  **スコア分布の spreader (Layer 5)**: `percentile.powf(0.3)` により、NABの「限定的な閾値探索範囲」においても、モデルの予測誤差がスコアに鋭敏に反映されるようになりました。

## 3. 次の課題: SOTA越えへのロードマップ

### 3.1 Standard スコアの不足原因
現在、Low FN/FP に比べて Standard スコアが伸び悩んでいる原因は、**「微細な異常の見逃し」** にあります。現在の単層 SSM (64次元) では、タクシーや Twitter データに見られる複雑な周期パターンの「わずかな乖離」をまだノイズとして処理してしまう傾向があります。

### 3.2 具体的対策 (Phase 4-5)

1.  **Multi-Scale Stack**:
    - 第1層: 短期的変化（$dt = 1$）に敏感な高解像度 SSM。
    - 第2層: 数百ステップ単位の季節性を捉える低解像度 SSM。
2.  **時刻情報の埋め込み**:
    - 平日/週末、日中/深夜のバイアスをモデルが「予測可能」なものとして学習させ、周期性異常に対する偽陽性をさらに削減します。
3.  **アクション端子の活用**:
    - `LatentPredictor` の `action` 入力に、外部イベント情報やラグ特徴量を注入します。
