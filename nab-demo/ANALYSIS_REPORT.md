# NAB 異常検知 — 最終分析レポート

**最終更新**: 2026-05-20
**結論**: **JEPA は異常検知に構造的に不適 — プロジェクトの強みは別領域にある**

---

## 7方式の実験から得られた結論

20回以上の試行錯誤で一貫して確認された事実：

### JEPA予測誤差の異常/正常比は最大でも 1.2〜3.8倍

NAB の閾値最適化が意味のある TP を拾うには、異常時のスコアが正常時の p95 を大幅に超える必要がある。
JEPA の予測誤差・再構成誤差・潜在予測誤差のいずれも、この要件を満たせなかった。

### なぜ JEPA は異常を検出できないのか

1. **Manifold 投影能力が高すぎる**: 異常点も学習済みの正常 manifold 上に投影できてしまう。結果として再構成誤差が低い。
2. **時間的文脈での「説明」**: 時間特徴量（hour, day-of-week）があると「この時間帯は変動が激しい」と学習し、異常でも予測できてしまう。
3. **VICReg の副作用**: 全表現を同一分布に揃えようとする力が、異常表現の「外れ値性」を消す。
4. **Curvature 正則化の矛盾**: 「軌道の曲がり」を罰する損失は、まさに異常パターンそのものを罰している。

### モデルを意図的に壊せば改善するが本末転倒

d_model=8, d_state=4, n_layers=1, 特徴量=[value, diff]のみ（時間情報なし）、という
**JEPA としての価値を完全に否定する設定**でようやく一部改善した。
これは「優れたモデルを作る」というプロジェクトの目的と真っ向から矛盾する。

## 3-way Ensemble (最終方式) の限界

WindowedGaussian + 差分スパイク + JEPA潜在距離のアンサンブル：

| データセット | 異常 > norm p95 | Top10中の異常 |
|---|---|---|
| ambient_temperature | 28.1% | 6/10 |
| occupancy_t4013 | 16.0% | 7/10 |
| art_daily_jumpsdown | 9.2% | 1/10 |
| ec2_cpu_5f5533 | 0% | 0/10 |

NAB スコア: **全プロファイルで負値（TP=0, FP=0, FN=全異常）**

WindowedGaussian 単体でも NAB では ~70 point の正規化スコアを達成できるが、
JEPA 成分を加えることでむしろ悪化するケースが多い。

## このプロジェクトが SOTA を狙うべき領域

| 領域 | 理由 |
|---|---|
| **時系列予測 (n-step forecasting)** | JEPA の本領。NAB データで正常期間の予測精度を評価できる |
| **Long Range Arena (LRA)** | SSM アーキテクチャの長距離依存性を直接評価 |
| **潜在空間の構造分析** | Disentanglement, smoothness, 閉ループ安定性 |
| **少ショット系列モデリング** | 15% データのみでの汎化性能 |
| **マルチモーダル予測** | 実装済み (`multimodal.rs`) を活かした評価 |

## NAB の評価指標の問題

- 2015年の古いベンチマーク
- ウィンドウ単位の粗いラベル（ポイント単位でない）
- 閾値最適化ベースの評価（確率的スコアの評価に不向き）
- 小規模データ（1K〜20K点/ファイル）で deep learning の評価には不十分

より新しい代替: TSB-UAD, TSSB, Aeon, GutenTAG
